## PoC 기획서: 문서 검열/수정 제출 게임

### 1. 핵심 컨셉

플레이어는 하나의 정보 문서를 받고, 각 단락을 **마스킹하거나 수정**한다.
수정된 문서는 여러 제출 대상 중 하나에게 제출할 수 있으며, 제출 대상의 성향에 따라 평가와 평판 변화가 달라진다.

---

### 2. 기본 플레이 흐름

1. 문서 확인
2. 단락별 정보 확인
3. 단락을 그대로 두거나, 마스킹하거나, 수정
4. 제출 대상 선택
5. 결과 확인
6. 평판 변화 발생

---

### 3. 문서 구성

PoC에서는 **문서 1개**만 사용한다.

문서는 여러 개의 단락으로 구성된다.
각 단락은 플레이어가 개별적으로 조작할 수 있다.

단락마다 최소한 다음 정보가 있다.

| 항목         | 의미                    |
| ---------- | --------------------- |
| **정보 가치**  | 이 단락이 문서에서 얼마나 중요한가   |
| **정보 무결성** | 수정 후 핵심 의미가 얼마나 보존됐는가 |

---

### 4. 플레이어 행동

단락마다 다음 행동을 할 수 있다.

| 행동      | 설명               |
| ------- | ---------------- |
| **유지**  | 원문 그대로 둔다        |
| **마스킹** | 일부 또는 전체 내용을 가린다 |
| **수정**  | 내용을 다른 표현으로 바꾼다  |

각 행동은 단락의 **정보 무결성**에 영향을 준다.

예시:

| 행동       | 정보 무결성 변화   |
| -------- | ----------- |
| 유지       | 높음          |
| 부분 마스킹   | 일부 감소       |
| 전체 마스킹   | 크게 감소       |
| 의미 유지 수정 | 약간 감소 또는 유지 |
| 의미 왜곡 수정 | 크게 감소       |

---

### 5. 제출 대상

플레이어는 수정한 문서를 여러 대상 중 하나에게 제출한다.

제출 대상은 각자 원하는 정보 상태가 다르다.

예시:

| 제출 대상  | 선호                            |
| ------ | ----------------------------- |
| 진실 추구자 | 정보 가치가 높고 무결성이 높은 문서 선호       |
| 검열 기관  | 민감한 정보의 무결성이 낮은 문서 선호         |
| 언론사    | 핵심 정보는 남아있지만 공개 가능한 수준의 문서 선호 |
| 내부 고발자 | 최대한 원문에 가까운 문서 선호             |

---

### 6. 평가 기준

제출 결과는 크게 두 요소를 본다.

| 기준         | 설명                |
| ---------- | ----------------- |
| **정보 가치**  | 중요한 단락을 어떻게 처리했는가 |
| **정보 무결성** | 핵심 의미가 얼마나 남아있는가  |

같은 문서라도 제출 대상에 따라 평가가 달라진다.

예시:

* 중요한 정보를 그대로 제출하면 진실 추구자 평판 상승
* 중요한 정보를 검열하면 검열 기관 평판 상승
* 중요하지 않은 정보를 수정해도 영향은 작음
* 중요한 정보를 어색하게 왜곡하면 대부분의 대상에게 불리함

---

### 7. 평판 시스템

각 제출 대상은 플레이어에 대한 **평판**을 가진다.

문서를 제출하면 선택한 대상의 평판이 증가하거나 감소한다.
필요하다면 다른 대상의 평판에도 간접 영향을 줄 수 있다.

예시:

| 결과                 | 평판 변화        |
| ------------------ | ------------ |
| 대상의 선호에 맞는 문서 제출   | 평판 상승        |
| 대상의 선호와 반대되는 문서 제출 | 평판 하락        |
| 핵심 정보 손상           | 특정 대상에게 큰 하락 |
| 위험 정보 은폐           | 특정 대상에게 상승   |

---

### 8. PoC 범위

PoC에서는 아래만 구현한다.

* 문서 1개
* 단락 여러 개
* 단락별 정보 가치
* 단락별 정보 무결성
* 단락별 유지 / 마스킹 / 수정
* 제출 대상 여러 명
* 제출 결과에 따른 평판 증감

---

### 9. PoC에서 제외할 요소

초기 PoC에서는 아래는 제외한다.

* 문서 자동 생성
* 자유 텍스트 수정
* 복잡한 문장 자연스러움 판정
* 정보 괴리감 / 어색함 수치화
* 다중 문서 시스템
* 장기 캠페인 구조

어색함이나 모순은 수치로 만들지 말고, PoC에서는 특정 수정 선택지에 붙는 고정 패널티 정도로만 처리한다.

---

### 10. 핵심 확인 포인트

이 PoC에서 검증할 것은 하나다.

> 플레이어가 “어떤 정보를 얼마나 남기고, 누구에게 제출할지” 고민하는 재미가 있는가?

이 재미가 확인되면 이후에 문서 수, 제출 대상, 평판 관계, 수정 방식, 문장 자연스러움 판정을 확장한다.
